# **🧪 深入了解 Jekyll：GitHub Pages 的幕後功臣**

如果你把建立網站比喻成蓋房子，傳統動態網站（如 WordPress）是現場施工；而 **Jekyll** 則是「預鑄屋」——它在工廠（你的電腦或 GitHub 伺服器）先蓋好，再直接送到目的地（瀏覽器）。

## **🚀 Jekyll 的核心作用**

### **1\. 將 Markdown 轉換為 HTML**

這是它最基本的功能。開發者只需要專注於撰寫內容（.md），Jekyll 會自動套用你指定的版面配置，並將其轉換成網頁檔案（.html）。

### **2\. 內容與樣式的完美分離**

* **內容**：存放在 Markdown 檔案中。  
* **樣式**：存放在主題（Themes）或 CSS 中。  
* **橋樑**：透過 \_config.yml 進行全站設定。

### **3\. 自動化網頁建置**

Jekyll 會自動掃描你的資料夾，處理分類、標籤、分頁，並生成一個完整的網站目錄結構。

## **🛠️ Jekyll 如何運作？**

Jekyll 的工作流程可以用一個簡單的公式來表示：

**Markdown (內容) \+ Liquid (邏輯) \+ Templates (版型) \= Static Website (靜態網站)**

* **Front Matter (開頭配置)**：在檔案最上方的 \--- 區塊，告訴 Jekyll 這個頁面要用什麼版型、標題是什麼。  
* **Liquid 引擎**：這是一種標籤語言（例如 {{ content }}），讓靜態檔案也能有簡單的邏輯處理（如：列出所有文章列表）。

## **💡 為什麼在 GitHub Pages 使用 Jekyll？**

1. **零成本代管**：GitHub Pages 內建支援 Jekyll，你不需要付錢租伺服器或資料庫。  
2. **極速效能**：因為產出的是純靜態檔案，瀏覽器讀取速度極快。  
3. **安全性高**：沒有資料庫，所以沒有 SQL 注入攻擊的風險。  
4. **版本控制**：你的整個網站內容都在 Git 的掌握之下，隨時可以還原舊版。

## **📊 運作示意圖**

1. **你 Push 代碼**：包含 .md 和 \_config.yml。  
2. **GitHub Actions 啟動**：在後端跑 Jekyll 編譯。  
3. **產出靜態檔**：存放在隱形的伺服器空間。  
4. **讀者瀏覽**：直接讀取編譯好的 HTML。
